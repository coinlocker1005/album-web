本稿では、部門アルバムウェブアプリをデプロイする手順を示します。

1 デプロイ先接続情報

サーバ名称 部門サーバ
IPアドレス 172.19.86.21
接続ユーザ docker
認証 公開鍵認証(*)

(*)公開鍵は別途連携されます。

2 開発~デプロイするまで

リリースすることになったタイミングで、`mvn package`コマンドでアプリケーションのjarファイルを作成してください (バージョニング等、リリース対象アプリの特定手順は、本稿では扱いません)。 作成されたjarファイルは、`<AppRoot>/target/album-web-<AppVersion>.jar`という名前になっています。 (例: album-web-0.0.0-SNAPSHOT.jar)

3 デプロイ

3.1 資材の配置

  • WinSCP等を利用して、以下のディレクトリに資材 (jarファイル) を配置します。
/var/opt/album-web/build/

3.2 ディレクトリの移動

  • Tera Term等を使用して部門サーバに接続し、以下のディレクトリに移動します。
cd /var/opt/album-web

3.3 デプロイシェルの実行

  • 部門サーバで、以下のコマンドを実行します。
/var/opt/deploy.bash
  • 画面の指示に従って、必要事項を入力します。

3.4 状態確認

  • 下記手順により、状態確認を行います。

4 フォールバック

4.1 ディレクトリの移動

  • 以下のディレクトリに移動します。
cd /var/opt/album-web

4.2 フォールバック先イメージの特定

  • 以下のコマンドを実行して、部門サーバ上に存在するアプリケーションイメージの一覧を表示します。
docker images | grep -e "^album-web" -e "^REPOSITORY"

REPOSITORY               TAG                 IMAGE ID            CREATED             SIZE
album-web                0.0.1-SNAPSHOT      3e3d756d86d3        About an hour ago   662MB
  • 上記の一覧より、フォールバック先イメージのバージョンを特定します(「TAG」と書いてある箇所がバージョンです)。

4.3 フォールバック

  • 以下のコマンドを実行して、稼働しているアプリケーションが存在する場合は停止させます。
docker-compose down
  • 以下のコマンドを実行して、フォールバック先アプリケーションイメージを再デプロイします。なお、<VERSION>は前の手順で特定したバージョンで置き換えます。
APPVERSION=<VERSION> docker-compose up -d

4.4 状態確認

  • 下記手順により、状態確認を行います。

5 状態確認

5.1 ディレクトリの移動

  • 以下のディレクトリに移動します。
cd /var/opt/album-web

5.2 コンソール上からの確認

  • 部門サーバ上で、以下のコマンドを実行します。 コマンド出力結果が、以下のようになっていることを確認します。(省略されている箇所以外、正確に同じであることを確認します。)
docker-compose ps 2> /dev/null

Name              Command               State            Ports
------------------------------------------------------------------------
app    sh -c java -Dspring.profil ...   Up      127.0.0.1:9000->9000/tcp
db     docker-entrypoint.sh postgres    Up      5432/tcp
  • 部門サーバ上で、以下のコマンドを実行します。 コマンド出力結果が、以下のようになっていることを確認します。(Repositoryがalbum-webであるレコードのTagが、指定したバージョンであることを確認します。)
docker-compose images 2> /dev/null

Container   Repository        Tag           Image Id      Size
---------------------------------------------------------------
app         album-web    0.0.1-SNAPSHOT   7a07ac86f0b2   631 MB
db          postgres     10               57f72dd0497d   219 MB

5.3 Webブラウザからの確認

  • Tetra上で稼働している端末より、webブラウザのアドレスバーに以下を入力します。
https://172.19.86.21/album
  • 証明書に関する警告が表示された場合は、警告を無視して進みます（部門サーバで自己署名証明書を使用しているためであり、今回は問題ありません）。
  • 部門アルバムサイトのトップページが表示されることを確認します。

6 後片付け

  • デプロイシェルの実行が完了したら、必要に応じて配置した資材を削除します。
  • 以下のコマンドを実行して、不要なアプリケーションイメージを特定します (IMAGE IDを調べます)。 今回デプロイしたイメージ以外はすべて不要です。
docker images | grep -e "^album-web" -e "^REPOSITORY"

REPOSITORY               TAG                 IMAGE ID            CREATED             SIZE
album-web                0.0.1-SNAPSHOT      3e3d756d86d3        About an hour ago   662MB
  • 以下のコマンドを実行して、不要なアプリケーションイメージを削除します。(複数のIMAGE IDを指定できます)
docker rmi <IMAGE ID> <IMAGE ID> ...
